## Some notes on the confidence score (37:51)

We need to define this. It's called confidence and should reflect, as I said, the confidence of your model that whatever box it proposes contains an object. However, it should also indicate how well the predicted box matches the ground truth. This can be slightly offset. You don't necessarily need to contain the whole object; you can miss it slightly. This should be reflected in the confidence. To achieve this, we define the confidence as the probability that there is an object multiplied by the intersection over union. Intersection over union is simply the ratio of the overlapping area to the total area of the union. It's straightforward: if we have two objects in a box, a generic prediction might miss it slightly or entirely, depending on the overlap. The overlap is the shared pixels, while the union is the total area of the two boxes. If the entire object fits within the predicted box, the confidence is high. Conversely, if the predicted box misses the object, the confidence is zero. In essence, this confidence not only detects the presence of an object but also localizes it within the image.

## Choosing a suitable loss (43:24)

So that's coming back to the problem and how we set up a suitable loss for this task. Recall we have three things to predict: classify things, place boxes around all objects in the image, and specify a number indicating the model's confidence in containing an object. For the classification part, we use categorical cross entropy. The box placement can be any method; a common choice is the sum of squared differences of the box coordinates. True bounding box coordinates consist of four numbers, and we aim to predict these. Squaring the difference between the true and predicted coordinates makes sense as a loss function because large discrepancies result in large values. This is similar to mean squared error.

For the bounding box confidence, defining a loss is more complex but manageable. Confidence is defined as the probability of an object multiplied by the intersection over union (IoU) between the prediction and ground truth. In simpler terms, does the box contain an object, and how well does the prediction match the ground truth? This confidence is learned during training, starting from poor performance and improving over time.

## Overlapping bounding boxes (48:33)

That's essentially it. In object detection, you process an image with a bunch of objects and predict boxes around each one. Using a specific loss function helps the model perform this task simultaneously. However, there's a slight complication: if an object like a cat has multiple bounding boxes, you might end up with several overlapping boxes. For instance, the red box covers most of the cat, while the green and blue ones cover partial areas. If the learned confidences for these boxes are 0.9, 0.7, and 0.35 respectively, you want the model to recognize a single cat rather than three. This is straightforward; if there's overlap, you discard the lowest-confidence boxes and keep the largest one. This logic applies during the prediction stage after training the model.

However, handling cases where multiple cats overlap can be challenging. Some object detection algorithms can manage this, while others cannot. The difficulty often depends on the threshold setting, which is a free parameter you can adjust based on practical performance. 

This limitation means that if you have multiple objects close together, such as seven birds in a small area, the model might struggle to distinguish them. To address this, you could increase the cell size, making the problem more complex.
