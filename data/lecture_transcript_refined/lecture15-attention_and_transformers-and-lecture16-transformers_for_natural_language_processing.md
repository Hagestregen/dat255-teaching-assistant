## The Transformer architecture (66:21)

That is the main thing. However, we still need to incorporate some elements before we reach what we refer to as the transformer. Attention serves as the mechanism, while the transformer represents the complete model architecture capable of achieving excellent results through the use of the attention concept. Let's now examine the first block of the flowcharts derived from the article introducing this topic. Additionally, we can identify several recognizable features from previous discussions. On this flowchart, data enters, followed by multi-head attention. There are also shortcut connections, reminiscent of residual connections in computer vision, which we commonly refer to as skip connections. These shortcuts are beneficial because they allow the model to continue training even if certain parts of the network fail to learn or if gradient issues arise. These shortcuts are crucial for maintaining the flow of information throughout the model. The outputs from the attention mechanism are then fed into a dense layer, where another residual connection is applied. The final output maintains the same dimensions, allowing for multiple iterations if necessary. This approach, as described in the original paper, involves attention, residual connections, and additional dense layers. Thus, we have two distinct areas for learning: the attention mechanism processes contextual information, while the subsequent dense layers perform pattern recognition and refinement. We can repeat this process as many times as required. For GPT-2, this process occurs approximately eleven times. There is one aspect we haven’t discussed yet. If we aim for classification, this method works well, enabling us to understand the relationships between different words. However, we will eventually need to generate text using this system. Additionally, we must consider that text is inherently ordered, albeit not strictly so, with words appearing in a specific sequence. Shuffling the input words disrupts the intended meaning, as the same word pairings produce identical scores regardless of their order. This poses a challenge since language naturally follows a sequential structure. To address this, we can incorporate the positional information of each word during the embedding phase. By embedding both the token and its position, we ensure that the model takes into account the sequence of words. One common approach is to embed the positions alongside the tokens, using indices that correspond to the word’s position in the sequence. This ensures that the model considers the relative order of words, thereby preserving the sequential nature of language.

## Positional Encoding (73:30)

Food was good. For each of these tokens, we get a vector, right? So 'food' maps to one vector, and 'position' maps to another. We'll do the same for position. Position one maps to this embedding vector, and position two maps to this other vector. Finally, we can sum them up. This gives us a final embedding that represents a point in the embedding space with both meaning and position. Adding vectors maintains the same dimensionality, so there's no change there. To be honest, it's a bit of a hacky solution, but it works. Essentially, we're mixing the meanings of the words and their positions. This results in a new vector that encodes both meaning and position. 

Additionally, if you're reading the original paper, they use a method called sinusoidal encoding. This involves creating vectors using sine waves that are shifted in phase, making them orthogonal at different positions. Introducing different frequencies creates vectors that vary based on their position. While this might seem a bit unconventional, it effectively encodes both the meaning and position of the tokens. Both methods—our approach and the sinusoidal encoding—can be used to encode both the meaning and position of tokens. 

This allows us to create meaningful combinations in the embedding space or latent space, such as 'not good' being different from 'good'. This approach scales well for any length or embedding dimension, provided the hardware can handle it.
