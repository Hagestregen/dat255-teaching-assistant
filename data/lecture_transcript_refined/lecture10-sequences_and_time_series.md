## Recurrent neural networks (RNNs) (23:06)

A cool method for doing so, which we called recurrent neural networks, are now kind of special because they have loops. They don't have a straight line of computation anymore. Instead, we take the output from one time step and feed it into the next time step. These networks have two inputs: the data itself and our previous prediction. However, they introduce a form of memory periodically. Each time we move to the next time step, we remember what happened at the previous time step. We can also remember even further back. One way to illustrate this is to write this for multiple time steps in a figure. To do this, we need to unroll the time dimension. Here, we see input at \( t-1 \). We generate an output as usual. But we also store this output as a state and place it in the box at time step \( t \). This process continues, taking input data and the state from the previous step. If we start at the beginning of the data, we use zero as the initial state. This serves as the memory. At each time step, we can make a prediction based on both the current data and the state from the previous step. For a clearer visualization, consider having several nodes within one such block. Essentially, a simple RNN, which we call it, is just a dense layer with a loop. Implementing this, we need an extra set of weights. We multiply these weights with the states and the original weights with the inputs, then sum them up. This is very similar to what we saw in dense layers before, except we add an extra set of weights for multiplying states. This technique is used in Keras and is called a simple RNN. However, there could be more complex versions, such as a complicated RNN, which we will explore later. Before diving into these complexities, let's briefly discuss time series.

## Intermezzo: Autoregressive models (26:33)

We go to the most complicated type of layer we will use, which is looking at old school statistics. Before deep learning, we had time series. What did people do then to make predictions? They used what’s called autoregressive models. In plain language, this means predicting based on the value from the same data at the previous time step. For this week’s notebook, we want to predict the number of passengers taking public transport in Chicago, as we have data for this city. It seems there’s a pattern, with higher usage on weekdays and lower on weekends. This pattern is known as seasonality, meaning it changes with a regular pattern over time. 

We can recognize this pattern by plotting the autocorrelation function, which shows how each measurement correlates with the previous day. For example, if today is Tuesday, the correlation with Tuesday of the previous week is high, while the correlation with Sunday of the previous week is low. This weekly pattern repeats every seven days, indicating a consistent cycle. 

By adding terms for each day of the week, we create a statistical model with parameters that need to be fitted. Essentially, we’re remembering what happened in the past to make predictions about the future. For instance, if today is Tuesday and the parameter for Tuesday of the previous week is high, it suggests a strong correlation, whereas a low parameter for Sunday indicates less correlation. 

Statisticians often refer to these models as ARMA (Autoregressive Moving Average) models, which include seasonality. Deep learning, while powerful, can sometimes struggle with time series data, making traditional statistical methods a useful baseline. Even a simple model like predicting today’s value based on yesterday’s value can perform surprisingly well, especially compared to deep learning approaches that often yield poor results. Therefore, for time series prediction tasks, using traditional statistical tools as a starting point can be beneficial.

Recurrent dropouts can be particularly useful for RNNs, especially when dealing with overfitting issues.

## Bonus trick #2: bidirectional RNNs (67:09)

These time series, whether for prediction or classification, include an interesting implementation in Keras known as bi-directional recurrent neural networks. One of our assumptions for time series data was that recent data points are likely the most important due to auto-correlation. Additionally, we need to retain some information from the past to maintain context. Ordering by time or sequence is useful as it aligns with our assumption. However, in some sequences, the order might be important but not critical. For the Google Translate task, we could consider a sequence-to-sequence approach where the input sequence is reversed and fed into the RNN. This allows the RNN to process the sequence in a different order, potentially making predictions based on earlier parts of the sequence. The bi-directional layer in Keras can handle this reordering, operating on LSTM layers or any other sequence-processing layer. While this approach might seem unconventional, it often works well. 

In the book, the processing direction is described as output data, but it should be outputs data. The bi-directional layer handles the reordering, while the input layer processes the actual sequence. The bi-directional layer itself doesn’t have any weights; it simply rearranges the sequence. This technique can be particularly useful when challenging the assumption that recent data is the most important. 

For ensemble learning, we can combine predictions from different models, treating each as a mini-ensemble within the network. If we predict an entire sentence rather than just the next step, bi-directional processing can be beneficial. Tonight, you can experiment with coding this up, though it might not be overly complex.
