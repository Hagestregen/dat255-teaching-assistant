## Image segmentation (16:18)

But that was maybe one example. So we're supposed to do definitions. There are different types of image segmentation you can perform. Essentially, they are based on the same principles. We divide it into what you are supposed to do with your outputs. One type is semantic segmentation, where semantics refer to what something means or what an object represents. For instance, if there are multiple cats in an image, I can classify them as cats and everything else as background. Instance segmentation, on the other hand, focuses on finding and distinguishing individual instances. If there are multiple cats, I can differentiate them and apply masks to each cat, separating them into distinct instances. Panoptic segmentation combines both categories; it outputs information about the category and the specific instance. For each pixel in the image, a prediction is made. ## Transposed convolution (30:30)

Transposed convolution is essentially convolution in reverse, particularly when considering dimensions. We had kernels applied to different spots; for each spot, we multiplied and summed, resulting in one pixel. Now, we want to do the opposite—expand one pixel into potentially four pixels. Instead of simply pasting information, we can use a clever method called transposed convolution. 

For instance, if our input pixel is zero, we multiply it by each pixel in the kernel. However, unlike traditional convolution, we do not sum these products; instead, we place each result at different spots. For example, zero times zero remains zero, zero times one is zero, and so on. This results in a pattern that expands from a 2x2 pixel area to a 3x3 area. Thus, we upscale the image in a smart way, leveraging a kernel learned during training. This process reverses the encoding step, allowing us to decode a smaller, compressed image into a larger, expanded version. 

We can visualize this trick and refer to books or YouTube tutorials for further details. To assemble the encoding and decoding process, we repeat this transposed convolution six times, ensuring stride and kernel size match. This results in compressing and then decompressing the image. Finally, our model outputs one of three classes: cats, backgrounds, or contours. We use softmax to ensure the output sums to one. The model summary shows the input size of 200x200, followed by a compressed section of 25x25 pixels, expanding to 50x50, and finally returning to 200x200. The model outputs three classes: foreground, background, and contour. This model meets our requirements, maintaining the same dimensions for outputs as inputs and utilizing a compression layer to store learned information.
