## Getting text into our model (16:30)

Step is of course getting text into the model. So we know neural networks have weights and biases, and these parameters are all numbers. Words cannot be multiplied with numbers; we need everything to be numbers. Better numbers lead to better results. A typical process for text classification involves starting with the text itself. Standardization, which reduces various spellings into consistent terms, such as converting uppercase letters to lowercase and removing punctuation, is crucial. For example, if "One" is a word, we convert it to "one." Tokenization follows, turning words into tokens—units of meaning. In simple cases, each word is a token. We split text using whitespace, creating a straightforward sequence of tokens. Later, we might use more sophisticated methods, but for now, we focus on basic tokens. Afterward, we assign indices to these tokens, making them numerical entities. For instance, "cat" could be token number 26. Typically, common words are assigned smaller indices, ensuring consistency. When we introduce an ordering, it becomes tricky, as words like "cat" and "Matt" might have the same index. To address this, we can use one-hot encoding, where each token is represented by a column indicating its presence. This creates a matrix showing the presence of different tokens. However, this method becomes inefficient with large dictionaries, leading to mostly zeros. To improve efficiency, we use embeddings, which represent words as vectors. This allows us to maintain the semantic relationships between words without needing a large number of columns. For simpler tasks, one-hot encoding might suffice, but for more complex models, embeddings provide a more effective solution.

## Text vectorisation (23:12)

Keras library, as we like. There's a Texts Vectorization layer that converts texts into integers. It does this by taking texts, features which can be words or tokens, and outputs a number for each token. This layer comes with options that you can modify. For instance, if your text typically has under 100 tokens, you can specify this number, and any token beyond 100 will be labeled as unknown. You can set this to the number of different words you have, or limit it to any number to create a smaller model. Another option is Standardization, which automatically removes punctuation and sets everything to lowercase. You can also provide your own function for more advanced regex operations. The output mode can be integer or one-hot encoding, depending on your needs. This layer is convenient because it can be part of an entire model, so you don't have to do this separately. If you want to learn the different words or tokens from your data, you can use the adapts function. This function doesn't fit parameters; instead, it adapts to your specific data set to determine the vocabulary. In the first notebook, we'll classify movie reviews using a dataset from IMDB, which maps text to either positive or negative reviews. When encoded, texts like "I am shocked shocked ends made" produce integers such as "shocked," which is not very common and thus has a high number. Decoding these numbers back gives us the standardized version, like "shocked." This process allows us to understand the text meaning. Often, we need to clean the data further, such as removing HTML tags. You can define your custom standardization to handle such cases.
