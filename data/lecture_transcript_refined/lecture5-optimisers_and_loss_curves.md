## AdaGrad (Adaptive Gradient) (32:24)

These tricks, let's do one trick as well. If you looked at the optimizer list in Keras, there were many with names starting with ada, such as ada grad, adam, and adam w. Many of these have funny names, often starting with ada for some reason. Let's explain what this means. It involves making an adaptive gradient, which means adjusting the learning rate differently for each parameter. If one parameter decreases very quickly, we can reduce the learning rate quickly to avoid overshooting. For parameters that decrease slowly, we can increase the learning rate slightly to speed up convergence. 

For instance, if I have two parameters, I can move along their respective distances. This approach leads to better performance on certain problems because different parameters affect the loss landscape differently. 

However, this method isn't widely used because it only works well for relatively simple problems. You can essentially forget about this method. Instead, there's a more sophisticated technique called RMSProp. The 'prop' in RMSProp stands for propagation, and 'RMS' refers to the root mean square of past gradients. This method considers how past gradients influence the current update, allowing for more nuanced adjustments to the learning rate. 

While this method is less sensitive to the learning rate, it may break down on complex problems. Therefore, combining this adaptive gradient method with other optimization techniques can lead to better results. RMSProp is generally recommended over this simpler method.
