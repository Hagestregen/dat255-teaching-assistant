## The activation function (45:54)

Convolutions were initially difficult functions, but now let's look at the simplest types of functions. In neural networks, an essential component is the activation function, which can appear in various forms. I claimed earlier that the key aspect is that it’s not a straight line; if it were, we could argue that adding more layers would result in a linear composition, making the overall model linear. While this argument holds, the point remains valid: to achieve more flexibility and complexity, we need to introduce non-linearity through the activation function.

There are several considerations for a good activation function. First, it must be non-linear. Second, for gradient descent to work, the function must be differentiable. As long as we can compute derivatives for most parts, it’s acceptable. Lastly, the activation function should activate, meaning it should turn on at some point and remain off for the rest of the range, effectively indicating yes or no.

One simple and effective activation function is the sigmoid, which ranges between 0 and 1. Although it might not look impressive on a scale, it ensures values can go as low as desired without reaching zero. Another popular choice is the hyperbolic tangent (tanh), which spans from -1 to 1. This function is widely used but may not be optimal in all cases.

The rectified linear unit (ReLU) is particularly notable. It consists of two straight-line segments: one for positive inputs and another for negative inputs. ReLU is easy to differentiate everywhere except at zero, where the derivative is undefined. Despite this, ReLU is highly effective and commonly used due to its simplicity and efficiency.

In summary, activation functions play a crucial role in neural networks by introducing non-linearity. Sigmoid and tanh are straightforward options, while ReLU offers a balance between simplicity and effectiveness. The choice of activation function depends on specific needs, but ReLU is often recommended as a good starting point.

## Finding optimal parameters (55:09)

Layers can be, for instance, all dense layers, with a vast number of these connections and numerous weights. However, for any proper deep learning model, there will be a significant number of parameters that need to be optimized simultaneously. This becomes complicated because each layer takes its input from others, whose parameters are adjusted based on the outputs of these layers. Consequently, many numbers need to be optimized at the same time. Tweaking a number in one layer affects all others, making it difficult to adjust one without considering the others. This issue was addressed in the early 1990s, sparking the deep learning revolution. The method used to optimize all these parameters is called backpropagation. ## Backpropagation (56:45)

Simplify the model a little bit, and the first step is to define these as my layers and nodes. I need to decide on the number of these layers, and each has a certain number of parameters. We choose a loss function that penalizes large errors and rewards small ones. This loss function is crucial because it measures the error and guides the optimization of the parameters.

Starting from the end, we can compute the derivatives, which we call gradients, in a high-dimensional space. We compute these gradients backward, starting from the loss and moving towards the input. During each iteration, we adjust the parameters based on the gradients, moving them towards the optimal solution. We continue this process, refining the parameters until the loss curve shows improvement.

## Dropout regularisation (81:15)

What we often want to do is this, and it’s a somewhat hacky approach but it works. You can develop a lot of theory about how it works. This method is called dropout. The basic idea is to randomly remove some nodes during training, thereby also removing parameters. If I have a densely connected network with many parameters and connections, I can randomly set some nodes to zero during training. The idea is that each remaining node is forced to learn information that it can’t rely on the neighboring nodes, which we call co-adaptation. During actual prediction, I run the entire network but randomly during training, I remove some nodes and then put them back, forcing each node to try and do something useful since it can’t just rely on a single input from these other nodes. This is a way of regularizing, helping to prevent overfitting, and it’s a common technique used in most large deep learning models. You can get used to it; just specify how many nodes to drop as a percentage, usually between 10% and 50%, rarely more than 50%.
