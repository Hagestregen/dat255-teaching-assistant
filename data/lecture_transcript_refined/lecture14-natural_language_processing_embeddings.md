## Embeddings (3:45)

These are the so-called embeddings. Previously, we discussed that we can represent categorical values using vectors. For instance, if I have different classes, I can map them onto an array of numbers. This is technically referred to as a vector. Given that we are dealing with texts, if I categorize different tokens as different words, the process of encoding this into embeddings involves taking this one. Instead of having a single column indicating whether we've seen a specific word, we now have numbers, which can have various qualities. These numbers are initialized randomly at the beginning, similar to how we initialize model parameters. The choice of how many random numbers to use is arbitrary and depends on the desired dimensionality of the embedding space. This dimensionality can range from a small number to a larger one, affecting both the amount of training data needed and the flexibility of the embedding space. Initializing these vectors randomly allows us to optimize them during training. Unlike one-hot encoding, which simply assigns binary values to each word, embeddings aim to learn meaningful vector representations for words. These vectors create a vector space where words are positioned in different directions, offering mathematical properties that facilitate visualizations and distance calculations. To compute the mapping from words to these numbers, we perform one-hot encoding followed by a dense layer with the `Keras.layers.embedding` layer, which handles the encoding efficiently. As a developer, the key steps involve determining the size of the vocabulary (number of input nodes) and the desired dimensionality of the embedding space (number of output nodes). For modern large language models, the embedding space often includes around 3000 to 4000 dimensions, depending on the complexity required. Initializing these vectors randomly and optimizing them through training yields meaningful embeddings that capture semantic relationships between words.

## Latent spaces (77:30)

That places us now in the position to make a definition of latent spaces. They are an abstract concept, but essentially, they are where our better numbers reside. In a model with different layers, inputs, hidden layers, and classification layers, we previously referred to the hidden layer as a somewhat misleading term. Let’s instead call it the latent feature space, as these are the numbers we can use to make better decisions rather than just the inputs. There are various methods to obtain the latent feature space. With deep learning, such spaces are often obtained automatically for free, one per layer. The key point is that we aim to improve these numbers sequentially. Thus, the latent spaces become progressively better as we move further along the model, becoming more specific. To save time and effort, we can use pre-trained models through transfer learning. Early in the model, these latent spaces tend to be quite general, suitable for multiple tasks. Towards the end, they become highly specific to a particular task. This is where we can make adjustments. Assuming a model trained on one thing has identified useful features for related tasks, we can apply this knowledge to new problems. For instance, if we want to detect cancer using x-ray images and have limited training data, we can train our model on a large dataset of x-ray images. The initial latent space, which extracts general x-ray features, can be reused. Only the final parameters need tuning. This approach allows us to solve our problem without overfitting, as we only modify a small portion of the model. Additionally, if we lack both training data and time, we can use open-source models that have already been pre-trained. By reusing these models, we can tackle different types of problems efficiently. This practice is particularly common for language models, which require extensive pre-training. Making a state-of-the-art language model typically involves significant computational resources and time. However, starting from a pre-trained model simplifies the process considerably.
